CHAPTER 0
アルゴリズムとフローチャート
プログラムを書く前に「手順の設計図」を理解する。
📌 この章の学習目標
  • 「アルゴリズム」とは何かを説明できる
  • フローチャートの主要な図形の意味を言える
  • プログラミング的思考の 3 要素（順次・分岐・反復）を DNCL のどの構文に対応するか言える

0-1　アルゴリズムとは

アルゴリズムとは、ある問題を解くための手順（ステップの列）のことです。プログラムはアルゴリズムを、コンピュータが理解できる言語で書いたものです。

例：「クラス全員の点数の平均を求める」アルゴリズム
① 合計を 0 にする（初期化）
② 全員分、点数を合計に加える（反復）
③ 合計 ÷ 人数 を計算する（計算）
④ 結果を表示する（出力）
自然言語で書いた手順がアルゴリズム。これをプログラムで表現する。

0-2　プログラミング的思考の 3 要素

どんなプログラムも、次の 3 種類の構造だけで表現できます。

構造意味DNCL の構文フローチャートの形
順次（Sequential）上から順に 1 行ずつ実行する代入・表示・演算長方形を縦に並べる
分岐（Selection）条件によって実行する処理を選ぶもし〜ならば〜を実行するひし形（条件）から分岐
反復（Iteration）条件が続く間、処理を繰り返すfor / 前判定 / 後判定ループひし形に戻る矢印
ポイント：「プログラミング的思考」とはこの 3 要素を組み合わせて問題解決の手順を設計する力のことです。プログラム言語が変わっても、この考え方は共通です。

0-3　フローチャートの記号

開始・終了
楕円（端子）
プログラムの始まりと終わり
処理
長方形（処理）
代入・計算・表示など
ひし形（条件）
はい／いいえで分岐

CHAPTER 1
変数と代入
データを名前のついた「箱」に入れて、後から取り出す仕組みを学ぶ。
📌 この章の学習目標
  • 「変数」とは何かを自分の言葉で説明できる
  • 代入記号 ← を使って変数を正しく宣言・更新できる
  • 変数の値を組み合わせた表示文（文字列 と 変数 を表示する）を書ける

1-1　変数とは何か

プログラムは数値や文字を計算・記憶・表示することで動きます。その「記憶」の役割を担うのが変数です。変数は、値を入れておける名前つきの箱だと考えてください。

変数名：合計
368
← 右の値を左の箱に入れる
変数「合計」に 368 を代入した状態。プログラムのどこからでも「合計」と書くと 368 として使える。

1-2　代入の書き方

構 文
変数名 ← 値や式

記号 ← は「右の値を左の変数に入れる」を意味します。等号 = ではないことに注意してください。

例 ― 変数に数値を代入する
得点 ← 85
クラス ← 3
平均 ← 得点 ÷ 2
「得点は、」 と 得点 を表示する
例 ― 変数の値を更新する
カウント ← 0
カウント ← カウント ＋ 1
カウント ← カウント ＋ 1
「カウント = 」 と カウント を表示する
→ 表示：カウント = 2
ポイント：右辺に自分自身が出てきてもよい。カウント ← カウント ＋ 1 は「現在の値に 1 を加えた結果で上書きする」という意味。

1-3　変数名のルール

ルール良い例悪い例
日本語でも英字でも使える得点、score（なし）
意味のある名前をつける合計、最大値a、x1（何の値か分からない）
注意：同じ変数名に何度でも代入できる。新しく代入すると古い値は消える。
補足（配列の命名について）：本アプリ「大富豪プログラミング」では、配列変数を大文字始まりで書く慣習を使っています（例：Scores, テン）。ただし、これは本アプリ固有の取り決めであり、共通テスト用プログラム表記（DNCL）の公式ルールではありません。共通テスト本番では命名規則に特に制限はないため、日本語の意味のある名前（例：点数配列、気温データ）を自由に使うことができます。

1-4　値の表示

構 文
式 を表示する
「文字列」 と 式 を表示する
「文字列1」 と 式1 と 「文字列2」 と 式2 を表示する
例 ― 文字列と変数を組み合わせて表示する
名前 ← 「山田」
点数 ← 92
「名前：」 と 名前 と 「　点数：」 と 点数 を表示する
→ 表示：名前：山田　点数：92
🎮 大富豪との接続：大富豪プログラミングでは「手札の枚数」「場のカードの強さ」「現在のスコア」など、すべてを変数で管理しています。たとえば cnt ← 0 でカウンタを初期化し、カードを 1 枚処理するたびに cnt ← cnt ＋ 1 で更新する操作は、基礎問題 b01〜b05 で繰り返し登場します。
✅ 自己チェック（第 1 章）
「変数は値を入れておく名前つきの箱」と自分の言葉で説明できる
x ← 5 の後 x ← x ＋ 3 を実行すると x が 8 になることを確認できた
「文字列 と 変数名 を表示する」の構文を書ける
DNCL の代入記号は ←（＝ではない）ことを説明できる
確認問題 ① 変数と代入
  1. 次のプログラムを実行すると、最終的な変数 x の値はいくつか。
    x ← 10 ／ x ← x ＋ 5 ／ x ← x × 2  → 
  2. 「今日の気温は〇〇℃です」（〇〇は変数 気温）と表示する 1 行を書け。
  3. a ← 3 の後に a ← a × a を実行すると a は になる。

CHAPTER 2
演算子
計算・比較・論理の 3 種類の演算子を使いこなす。
📌 この章の学習目標
  • 算術・比較・論理の 3 種類の演算子を区別して使い分けられる
  • DNCL の ÷ が整数割り算（小数点以下切り捨て）であることを説明できる
  • かつ・または・でない を使った複合条件式を正しく書ける

2-1　算術演算子

演算子意味例（a=7、b=2）結果
＋足し算a ＋ b9
－引き算a － b5
×掛け算a × b14
÷整数の割り算（小数点以下切り捨て）a ÷ b3（3.5 ではない）
％余り（モジュロ）a ％ b1
重要：÷ は整数の割り算です。7 ÷ 2 = 3（0.5 は捨てられる）。小数が必要な場面では先に掛けるなど工夫が必要です。
例 ― 消費税込み価格の計算
税抜 ← 980
税込 ← 税抜 × 110 ÷ 100
「税込価格は、」 と 税込 と 「円」 を表示する
→ 表示：税込価格は、1078円

2-2　比較演算子

条件を作るときに使います。結果は真（条件が成立する）か偽（成立しない）かのどちらかです。

演算子意味例結果
＝等しい5 ＝ 5真
≠等しくない5 ≠ 3真
＞より大きい5 ＞ 3真
＜より小さい5 ＜ 3偽
≧以上5 ≧ 5真
≦以下3 ≦ 5真
ミス注意：>（半角）と ＞（全角）は見た目が似ていますが、DNCL では全角の ＞ のみが正しい演算子です。半角を使うと誤動作します。

2-3　論理演算子

複数の条件を組み合わせます。

演算子意味真になる条件例
かつAND（両方）左も右も真のとき点数 ≧ 60 かつ 出席 ＝ 1
またはOR（どちらか）左か右の少なくとも一方が真のとき点数 ≧ 90 または 特別加点 ＝ 1
でないNOT（否定）元の条件が偽のとき点数 ＝ 0 でない
かつの真理値表： 真 かつ 真 → 真 ／ 真 かつ 偽 → 偽 ／ 偽 かつ 真 → 偽 ／ 偽 かつ 偽 → 偽
またはの真理値表：真 または 真 → 真 ／ 真 または 偽 → 真 ／ 偽 または 真 → 真 ／ 偽 または 偽 → 偽
例 ― 複合条件
年齢 ← 17
会員 ← 1
もし 年齢 ≧ 18 または 会員 ＝ 1 ならば
｜　「入場できます」 を表示する
を実行する
→ 年齢は 17 で条件を満たさないが、会員 = 1 で「または」が真 → 入場できます

2-4　演算子の優先順位

括弧 ( ) を使うと優先順位を明示できます。迷ったら括弧をつけましょう。

括弧なし（意図しない計算になる可能性）
2 ＋ 3 × 4　→　14（×が先）
括弧あり（意図を明確に）
(2 ＋ 3) × 4　→　20
🎮 大富豪との接続：カードゲームでは「場のカードより強いか（＞）」「同じ強さか（＝）」「革命中かつ手札に 4 枚揃いがあるか（かつ）」など、比較演算子と論理演算子が随所で使われます。基礎問題 b10〜b12 では比較演算子を実際に書いて動かします。
✅ 自己チェック（第 2 章）
7 ÷ 2 が 3（小数点以下切り捨て）になることを確認できた
7 ％ 2 が 1（余り）になることを確認できた
比較演算子（＞ ＜ ≧ ≦ ＝ ≠）がすべて全角であることを覚えた
かつ と または の真理値表を見ずに説明できる
確認問題 ② 演算子
  1. 17 ÷ 5 の結果は 、17 ％ 5 の結果は である。
  2. a ← 8 のとき、a ％ 2 ＝ 0 は 真 / 偽 のどちらか。
  3. 「18 歳以上、かつ、学生でない」を DNCL の条件式で書け（年齢・学生フラグ 0/1 を使う）。

CHAPTER 3
条件分岐
条件によって実行する処理を切り替える「もし〜ならば」の使い方。
📌 この章の学習目標
  • 「もし〜ならば〜を実行する」の構文を正確に書ける
  • if-else・if-elif-else の多分岐を正しい順序で組み立てられる
  • 入れ子（ネスト）した条件分岐を読んで実行結果を追える

3-1　もし〜ならば（if 文）

構 文
もし 条件 ならば
｜　条件が真のときの処理
を実行する

｜　（縦棒＋全角スペース）はブロックの内側を示すインデント記号です。「を実行する」で閉じます。条件が偽のときは何もせず次の行へ進みます。

例 ― 合格判定
点数 ← 72
もし 点数 ≧ 60 ならば
｜　「合格です」 を表示する
を実行する
「処理終了」 を表示する
→ 合格です
→ 処理終了

3-2　もし〜そうでなければ（if-else 文）

構 文
もし 条件 ならば
｜　条件が真のときの処理
を実行し，そうでなければ
｜　条件が偽のときの処理
を実行する
例 ― 合否を必ずどちらか表示する
点数 ← 48
もし 点数 ≧ 60 ならば
｜　「合格です」 を表示する
を実行し，そうでなければ
｜　「不合格です」 を表示する
を実行する
→ 不合格です

3-3　3 分岐以上（if-elif-else）

構 文（3 分岐の例）
もし 条件A ならば
｜　処理A
を実行し，そうでなければ もし 条件B ならば
｜　処理B
を実行し，そうでなければ
｜　処理C
を実行する

条件は上から順に判定されます。最初に真になった分岐が実行され、残りはスキップされます。

例 ― 成績評価（S/A/B/C）
点数 ← 78
もし 点数 ≧ 90 ならば
｜　「S」 を表示する
を実行し，そうでなければ もし 点数 ≧ 75 ならば
｜　「A」 を表示する
を実行し，そうでなければ もし 点数 ≧ 60 ならば
｜　「B」 を表示する
を実行し，そうでなければ
｜　「C」 を表示する
を実行する
→ A（78 は 90 未満・75 以上なので 2 番目の分岐）

3-4　入れ子（ネスト）

条件分岐の中にさらに条件分岐を書くことができます。

例 ― 会員かつ 18 歳以上のみ割引
もし 会員 ＝ 1 ならば
｜　もし 年齢 ≧ 18 ならば
｜　｜　「シニア会員割引」 を表示する
｜　を実行し，そうでなければ
｜　｜　「学生会員割引」 を表示する
｜　を実行する
を実行し，そうでなければ
｜　「通常料金」 を表示する
を実行する
ポイント：入れ子が深くなるほど ｜　 が増える。読みにくいと感じたら、条件を かつ でまとめられないか検討する。
🎮 大富豪との接続：「出したカードが場より強ければ出せる、そうでなければ出せない」はまさに if-else です。基礎問題 b10〜b13 では、カードの強さ比較・スート判定・役の判定など、複数条件の分岐を組み合わせる練習ができます。
✅ 自己チェック（第 3 章）
「もし〜ならば … を実行する」を「を実行する」で必ず閉じることを理解した
多分岐（if-elif-else）は上から順に判定され、最初に真になった分岐だけ実行されることを説明できる
点数が 90 以上 → S、75 以上 → A、60 以上 → B、それ以外 → C の 4 分岐プログラムを書ける
入れ子（ネスト）した分岐で縦棒 ｜　 の数がどう増えるかを説明できる
確認問題 ③ 条件分岐
  1. 気温 t が 0 未満なら「氷点下」、0 以上 25 未満なら「快適」、25 以上なら「暑い」と表示するプログラムを書け。
  2. 次のプログラムで x ← 15 のとき、何が表示されるか。
    もし x ％ 3 ＝ 0 かつ x ％ 5 ＝ 0 ならば「FizzBuzz」…（FizzBuzz の 3 分岐プログラム）

CHAPTER 4
for ループ
「〇回繰り返す」「1 から N まで順に処理する」ときの書き方。
📌 この章の学習目標
  • 「〇 を △ から □ まで 1 ずつ増やしながら，〜を繰り返す」の構文を正しく書ける
  • ループが何回実行されるかを計算できる
  • カウンタパターン（ループ前に 0 で初期化 → 条件が真なら ＋1）を自力で実装できる

4-1　for ループの構文

構 文
変数 を 初期値 から 終了値 まで 差分 ずつ増やしながら，
｜　繰り返す処理
を繰り返す

変数（カウンタ）が初期値から終了値まで差分ずつ増えるたびに、内側の処理が 1 回実行されます。

i を 1 から 5 まで 1 ずつ増やしながら の動き：
i=1
→
i=2
→
i=3
→
i=4
→
i=5
→ 終了
合計 5 回実行される。

4-2　基本例：1 から N の合計

例 ― 1 から 10 の合計を求める
合計 ← 0
i を 1 から 10 まで 1 ずつ増やしながら，
｜　合計 ← 合計 ＋ i
を繰り返す
「1から10の合計は、」 と 合計 を表示する
→ 1から10の合計は、55

4-3　差分を変える

偶数だけ表示（差分 = 2）
i を 2 から 10 まで 2 ずつ増やしながら，
｜　i を表示する
を繰り返す
→ 2 4 6 8 10
逆順（1 ずつ減らす）
i を 5 から 1 まで 1 ずつ減らしながら，
｜　i を表示する
を繰り返す
→ 5 4 3 2 1

例 ― 1〜20 の中から 3 の倍数を表示
i を 1 から 20 まで 1 ずつ増やしながら，
｜　もし i ％ 3 ＝ 0 ならば
｜　｜　i を表示する
｜　を実行する
を繰り返す
→ 3 6 9 12 15 18

4-5　カウンタパターン

ループ前に変数を 0 で初期化し、条件が真のたびに 1 を加えます。これは最重要パターンの 1 つです。

例 ― クラス 30 人の中で 80 点以上の人数を数える（点数配列 テン を使用）
人数 ← 0
i を 1 から 要素数[テン] まで 1 ずつ増やしながら，
｜　もし テン[i] ≧ 80 ならば
｜　｜　人数 ← 人数 ＋ 1
｜　を実行する

CHAPTER 5
前判定ループ（while）
「条件が続く間だけ繰り返す」。繰り返す回数が事前にわからないときに使う。
📌 この章の学習目標
  • 「〜 の間，〜を繰り返す」の構文を正確に書ける
  • for ループと前判定ループの使い分け基準を説明できる
  • 配列の中から条件を満たす最初の位置を探す「探索パターン」を実装できる

5-1　前判定ループの構文

構 文
条件式 の間，
｜　繰り返す処理
を繰り返す

ループの先頭で条件を判定します。条件が真なら処理を実行し、また先頭に戻ります。最初から偽なら1 回も実行されません。

条件判定 → 真 → 処理実行 → 条件判定 → 真 → 処理実行 → ⋯ → 偽 → ループ終了（次の行へ）
for ループと違い、条件が複雑なときや繰り返し回数が不定のときに使う。

5-2　基本例：一定条件まで増やす

例 ― x を 2 倍し続けて 100 を超えるまで
x ← 1
x ≦ 100 の間，
｜　x ← x × 2
を繰り返す
「100を超えた最初の値は、」 と x を表示する
→ 100を超えた最初の値は、128
注意：ループ内で条件に関わる変数（上の例では x）を必ず更新してください。更新を忘れると条件が永遠に真のまま＝無限ループになります。

5-3　探索パターン（前判定ループの典型）

「条件を満たす要素を配列の中から探す」ときに多く使われます。

例 ― 点数が 70 以上の最初の位置を探す（配列 テン を使用）
i ← 1
i ≦ 要素数[テン] かつ テン[i] ＜ 70 の間，
｜　i ← i ＋ 1
を繰り返す
もし i ≦ 要素数[テン] ならば
｜　「最初に70以上なのは」 と i と 「番目: 」 と テン[i] を表示する
を実行し，そうでなければ
｜　「70以上の人はいません」 を表示する
を実行する
ポイント：かつ の左に「配列の範囲内か」という条件を置くことで、配列の外を参照するエラーを防ぎます。ループを抜けた後の i が「見つかった位置」を示します。

状況使うループ

繰り返す回数が決まっていない（条件が成立する間だけ）前判定ループ
最初の要素から条件を満たすものを探す前判定ループ

CHAPTER 6
後判定ループ・中断する
「必ず 1 回は実行してから判定する」後判定と、ループを途中で抜ける中断する。
📌 この章の学習目標
  • 「繰り返し，〜を，〜になるまで実行する」の構文を書ける
  • 後判定ループが前判定と異なる点（必ず 1 回実行・止まる条件が逆）を説明できる

6-1　後判定ループの構文

構 文
繰り返し，
｜　繰り返す処理
を，条件式 になるまで実行する

処理を実行してから条件を判定します。条件が真（「なるまで」の条件が成立）になったら止まります。最低でも1 回は必ず実行されます。

前判定との比較

前判定後判定
条件判定処理の前処理の後
最低実行回数0 回1 回
止まる条件条件が偽条件が真
例 ― 最後の要素まで順に処理
i ← 0
繰り返し，
｜　i ← i ＋ 1
｜　テン[i] を表示する
を，i ＝ 要素数[テン] になるまで実行する
→ テン[1]〜テン[最後] を順に表示
止まる条件に注意：前判定は「条件が真の間続ける」ですが、後判定は「条件が真になったら止まる」と逆の意味です。混同しないように。

6-2　中断する（break）

構 文
（ループ内で）
中断する

ループの途中で即座にループを抜けます。「見つかった瞬間に処理を止めたい」ときに使います。

例 ― 配列の中で最初に 80 点以上の人を見つけたらすぐ終了
見つけた ← 0
i を 1 から 要素数[テン] まで 1 ずつ増やしながら，
｜　もし テン[i] ≧ 80 ならば
｜　｜　見つけた ← テン[i]
｜　｜　中断する
｜　を実行する
を繰り返す
もし 見つけた ＞ 0 ならば
｜　「最初の80点以上：」 と 見つけた を表示する
を実行する
パターン：見つけた ← 0 で初期化し、見つかったら値を代入して 中断する。ループ後に値が 0 かどうかで「見つかったか」を判定する。

関数
処理に名前をつけて再利用できる「関数」の定義と呼び出し方。
📌 この章の学習目標

  • 引数・返り値の概念を説明し、関数を正しく呼び出せる
  • 関数を使ってプログラムを部品化（モジュール化）する利点を説明できる

8-1　関数とは

同じような処理を何度も書くのは無駄です。処理に名前をつけて「関数」として定義しておけば、その名前を書くだけで何度でも呼び出せます。

関数を使わない場合：点数[1] ÷ 5、点数[2] ÷ 5、… と繰り返し書く
関数を使う場合：偏差(点数[1])、偏差(点数[2])、… と呼び出すだけ
複雑な処理を 1 か所にまとめることで、プログラムが読みやすく保守しやすくなる。

定義は呼び出しより前に書く：DNCL では関数を使う前に定義しておく必要があります。
