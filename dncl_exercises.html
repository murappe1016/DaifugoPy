ここも押さえておこう！ ― 複数代入（1行に並べる）・インクリメント構文
① 複数の代入を「，」で1行に並べる
DNCL では複数の代入文を「，」で区切って1行に書けます。公式仕様では 左から順に実行されます（「同時に」ではありません）。
例 ― 合計に加えてからカウンタを進める
goukei ← goukei ＋ x, x ← x ＋ 1
　左の goukei ← goukei ＋ x → 右の x ← x ＋ 1 の順に実行
値の交換に注意：a ← b, b ← a は交換になりません。左から順に実行され、 a ← b で a が b の値になった後、b ← a では a がすでに 更新済みなので、a も b も元の b の値になってしまいます。交換するには一時変数を使います： w ← a, a ← b, b ← w。
② インクリメント・デクリメント構文
DNCL には「変数を 1 増やす」を表す短縮構文があります。
インクリメントの 2 通りの書き方（どちらも同じ意味）
kaunto を 1 増やす            ← 短縮構文（DNCL 仕様）
kaunto ← kaunto ＋ 1         ← 通常の代入（どちらでも正解）
試験では「kosu を 1 増やす」のように出題されることがあります。 両方の書き方を読み書きできるようにしておきましょう。

ここも押さえておこう！ ― 除算の種類と試験での表記バリエーション
① 整数除算（÷）と実数除算（/）の違い
DNCL では演算子によって除算の結果が変わります。
演算子 意味 例
÷（全角） 整数除算（小数点以下切り捨て） 7 ÷ 2 → 3
/（半角） 実数除算（小数も出る） 7 / 2 → 3.5
共通テストでは ÷ が多く使われますが、/ で実数除算を示す問題も出ます。 問題の注記を必ず確認しましょう。
② 試験での表記バリエーション
共通テストは DNCL 仕様と完全に同じではない場合があります（仕様書にも明記されています）。
項目 公式仕様 試験での表記例
代入 ← =
if 文 もし 条件 ならば ｜　... を実行する もし 条件 ならば: ｜　...
while 文 条件 の間，｜　... を繰り返す 条件 の間繰り返す: ｜　...
表示 「...」 を表示する 表示する(「...」)
対策：表記が違っても「何をしているか」を読む力をつけることが大切です。アルゴリズムの意味を理解すれば表記の違いに惑わされません。
