0-3　大富豪のルール（アプリ版）

大富豪プログラミングは、トランプの大富豪（大貧民）を元にした簡略版ルールでカードを出します。「どのカードを出すか」をDNCLコードで書くのがこのアプリの目的です。

⚠️ 通常の大富豪とのちがい（アプリ独自のルール）

項目通常の大富豪このアプリ
プレイ人数3〜6人など1対1（自分 vs CPU）
出せる枚数1枚・2枚・3枚・4枚…シングル（1枚）またはペア（2枚）のみ
革命・8切りなどあり（ゲームによる）8切りあり（革命なし）

プログラミング学習に集中できるよう、ルールを絞っています。
※8切り（8を出すと場が流れる）はフリープレイで体験できます。問題の採点には影響しません。

カードの強さ（ランク）

カード345678910JQKA$
数値3456789101112131415

3が最も弱く、$ が最も強い。プログラムでは数値（3〜15）で扱います。

A と $ の正体：A はトランプの「1」、$ は「2」のカードです。大富豪では 1（A）と 2 が最強格なので、A を 14、2（$）を 15 という強さの数値で表します。「A は本当は 1 なのに K（13）より強い」のと同じ考え方です。このアプリにジョーカーはありません。

手札について

手札は弱い順に並んでいます。
例：手札が「5」「10」「K」の3枚のとき
 手札[1] = 5（最弱） 手札[2] = 10 手札[3] = 13（最強 = K）

添字（番号）は 1 から始まります。手札[0] は存在しません。

0-4　DNCLとコードの書き方

DNCLとは

DNCL（ディーエヌシーエル）は、大学入試センターが定めたプログラミング記述言語です。日本語に近い書き方でアルゴリズムを記述します。

DNCLの特徴：
・日本語の単語をそのまま使える（もし、の間、を繰り返すなど）
・記号は全角を使う（←、＋、＝、≧など）
・インデント（字下げ）に ｜　（全角パイプ＋スペース）を使う
・共通テスト（情報Ⅰ）で使われる擬似コード

コードを書く・実行する手順

  1. エディタのアクティブなセクション（明るく表示されている部分）をクリックして選択する
  2. DNCLコードを入力する
  3. 「▶ 実行」ボタンを押す
  4. 右側のテストケースが全部 🟢 緑になれば正解！
  5. エラーが出た場合はエディタ下の赤いメッセージを確認する
⚠️ 入力の注意点：
・←（代入記号）は半角の <- でも入力できます
・＋ － × ÷ ＝ ≧ ≦ ≠ ＞ ＜ などは全角を使います
・日本語入力（IME）をオフにしてから入力するとスムーズです

左の「基礎リファレンス」をフル活用しよう

📖 1-4　手札の枚数を調べる ―― 要素数[ ]

手札の枚数は問題によって違います（2枚のこともあれば4枚のことも）。そこで、実行時に枚数を調べる方法が必要です。

要素数[手札] は、今の手札の枚数を表します。

例：手札 = [3, 7, 10] のとき
 要素数[手札] = 3

手札は弱い順に並んでいるので、最強カードの添字 ＝ 手札の枚数 です。

コラム
変数は「状態を記憶する箱」―― 大富豪の外の世界

第1章では、大富豪の 手札[i] や n ← 要素数[手札] を通じて「変数」を学びました。実は、変数は大富豪だけでなく、あらゆるプログラムの基礎です。

🛒 例1：ネットショッピングの在庫管理

商品の残り個数を変数で管理します。

zaiko ← 100　　# 在庫100個からスタート zaiko ← zaiko － 1　# 1個売れた → 99個 zaiko ← zaiko － 1　# また1個売れた → 98個

大富豪では 手札[1]（最弱カード）を取り出すために配列と変数を使いました。ネットショッピングでも在庫数を「箱（変数）」に入れて、減らしながら管理しています。仕組みは同じです。

🎮 例2：ゲームのスコア管理

score ← 0　　　# ゲーム開始時のスコア score ← score ＋ 100　# 敵を倒した → +100点 score ← score ＋ 50　 # アイテム取得 → +50点

大富豪では n ← 要素数[手札] で「今の手札枚数」を記憶しました。ゲームのスコアも「今の点数」を変数に記憶して、加算・減算で更新します。

📊 まとめ：変数は「状態のスナップショット」

分野変数が記憶するもの
大富豪（このアプリ）手札枚数、場の強さ、添字 i など
ゲームスコア、残り時間、プレイヤー位置
通販サイト在庫数、カート内の合計金額
気象観測気温、湿度、今日の最高気温

どの分野でも「今の状態を名前のついた箱に入れておき、必要なときに取り出す」という使い方は同じです。

📖 2-1　計算する ―― 算術演算子

プログラムでは、数を使った計算ができます。大富豪でも「2枚のカードの合計」や「真ん中のカードの位置」を計算で求められます。計算には算術演算子という記号を使います。

このアプリで使う算術演算子（すべて全角）：
記号意味例結果
＋足し算3 ＋ 47
－引き算10 － 37
×かけ算3 × 412
÷整数の割り算（小数切り捨て）7 ÷ 23
％割り算の余り7 ％ 21

📖 2-2　÷ は「整数の割り算」

このアプリの ÷ は、ふつうの割り算とちがって小数点以下を切り捨てます。これを「整数除算」と呼びます。

整数除算の例（小数点以下は切り捨て）：
式結果ふつうの割り算なら
7 ÷ 233.5 → 切り捨てて 3
9 ÷ 333（割り切れる）
5 ÷ 222.5 → 切り捨てて 2

これは「真ん中のカードの位置」を求めるときに役立ちます。たとえば手札が3枚なら、真ん中は (3 ＋ 1) ÷ 2 ＝ 2 番目です。

📖 2-3　％ は「余り」 ―― 偶数・奇数の判定

％ は割り算の余りを求めます。特に 2 で割った余り は、偶数・奇数を見分けるのに使えます。

偶数・奇数の見分け方：
枚数 ％ 2 ＝ 0 → 余りが0 → 偶数（2, 4, 6 ...）
枚数 ％ 2 ＝ 1 → 余りが1 → 奇数（1, 3, 5 ...）
⚠️ 記号はすべて全角で入力します。半角の + / % ではエラーになります。左パネルのボタンをクリックして入力すると確実です。
📱 b07「先頭2枚の合計を計算して表示する」 に挑戦しましょう！

「÷」と「％」はセットで覚える ―― 時間・お金の計算

第2章では ÷（整数の割り算）と ％（余り）を学びました。この2つは、日常のプログラムでもセットで大活躍します。

⏰ 例1：秒を「分と秒」に変換する

byou ← 125　　# 125秒 fun ← byou ÷ 60　# 125 ÷ 60 = 2（分） amari ← byou ％ 60　# 125 ％ 60 = 5（秒） # → 「2分5秒」

÷ で「何分か」、％ で「あまり何秒か」が同時に求められます。大富豪で真ん中の位置を (n＋1)÷2 で出したのと同じ仕組みです。

💴 例2：おつりの硬貨を計算する

otsuri ← 370 hyaku ← otsuri ÷ 100　# 100円玉 3枚 nokori ← otsuri ％ 100　# 残り 70円

レジや自販機のおつり計算も、÷ と ％ の組み合わせでできています。

📖 3-1　もし〜ならば ―― 条件で処理を分ける

「場より強ければ出す、弱ければパスする」のように、状況に応じて処理を変えたいことがあります。これを条件分岐といいます。

基本の形（条件を満たすときだけ実行）：
もし 条件 ならば
｜　処理
を実行する

条件が成り立つ（真）ときだけ、字下げした処理が実行されます。

この流れはフローチャート（流れ図）で表すと分かりやすくなります。条件のところで道が分かれ、「はい（真）」のときだけ処理を行い、その後は合流して次の行へ進みます。

フローチャートで見ると：
　　↓
　< 条件は真？ >
　├ はい  → ［処理を行う］
　└ いいえ →（何もしない）
　　↓
　次の行へ進む

📖 3-2　そうでなければ ―― 2通りに分ける

もし〜ならば だけでは「条件が真のときに何かする」だけでした。そうでなければ を加えると、真のときと偽のときで必ずどちらか一方の処理を行えます。

2通りに分ける形：
もし 条件 ならば
｜　処理A（条件が真のとき）
を実行し，そうでなければ
｜　処理B（条件が偽のとき）
を実行する

📖 3-3　比較演算子

条件は、2つの値を比べて作ります。比較には次の記号（全角）を使います。

記号意味例
＞より大きい手札[1] ＞ 場の強さ
＜より小さい手札[1] ＜ 5
≧以上（大きいか等しい）要素数[手札] ≧ 3
≦以下（小さいか等しい）手札[1] ≦ 場の強さ
＝等しい手札[1] ＝ 手札[2]
≠等しくない手札[1] ≠ 手札[2]
⚠️ ≧ は >=、≦ は <= と入力すると変換されます。比較演算子は左パネルのボタンからの入力が確実です。

📖 3-4　かつ・または・でない ―― 論理演算子

複数の条件を組み合わせるには論理演算子を使います。

記号意味真になるとき
かつAND（両方）条件Aも条件Bも成り立つとき
またはOR（どちらか）条件Aか条件Bの少なくとも一方が成り立つとき
でないNOT（否定）条件が成り立たないとき
⚠️ でない を使うときは、否定したい条件を ( ) でくくると範囲がはっきりします。例：(手札[1] ≦ 場の強さ) でない
📱 b10「場より強ければ出す」 に挑戦しましょう！
◆
コラム
条件分岐は「if文」―― 世界中のアプリの心臓部

第3章で学んだ もし〜ならば は、英語では if（イフ）文と呼ばれ、あらゆるプログラミング言語にある最重要機能です。アプリの「判断」はすべてこれでできています。

🚦 例1：自動ドア・信号機

もし 人が近づいた ならば ｜　ドアを開ける を実行し，そうでなければ ｜　ドアを閉める を実行する

「センサーが反応したか？」という条件で動作を変えています。大富豪で「場より強いか？」で出すか決めたのと同じ構造です。

🔐 例2：ログイン画面

もし パスワード ＝ 正解 ならば ｜　ログイン成功 を実行し，そうでなければ ｜　エラーを表示 を実行する

「入力 ＝ 正解」という条件分岐です。＝ による比較は b12（ペア判定）でも使いました。

📊 まとめ：論理演算子は「条件の組み立て」

場面条件の例
会員 かつ 18歳以上かつ（両方必要）
現金 または クレジットまたは（どちらかでOK）

大富豪の戦略づくりも、アプリの判断も、すべて「条件をどう組み立てるか」で決まります。

CHAPTER 4
forループ
同じ処理のくり返し ・ カウンタ変数 ・ フラグ変数

📖 4-1　くり返し ―― forループ

手札を1枚ずつ全部調べたいとき、手札[1]、手札[2]、手札[3] …と1つずつ書くのは大変です。forループを使うと、同じ処理を「番号を変えながら」くり返せます。

forループの形：
i を 1 から 要素数[手札] まで 1 ずつ増やしながら，
｜　処理（ここで 手札[i] を使う）
を繰り返す

変数 i が 1, 2, 3 … と自動で変わり、そのたびに字下げした処理が実行されます。i をカウンタ変数と呼びます。

手札=[3,7,10] のときの動き：
i=1 → 手札[1]=3 を処理
i=2 → 手札[2]=7 を処理
i=3 → 手札[3]=10 を処理
（i が 要素数[手札]=3 を超えたら終了）

📖 4-2　数えるパターン ―― カウンタ変数

「条件を満たすものが何個あるか」を数えるには、数える用の変数を用意し、ループの前で 0 にしておきます（初期化）。条件を満たすたびに ＋1 します。

# 数える変数を0で初期化 cnt ← 0 # ループの中で条件を満たすたびに1増やす cnt ← cnt ＋ 1

📖 4-3　見つけるパターン ―― フラグ変数

「条件を満たすものを見つけたか？」を覚えておく変数をフラグ変数といいます。最初は 0（まだ無い）にしておき、見つかったら値を入れます。

⚠️ 初期化を忘れない：数える変数・フラグ変数は、ループに入る前に必ず 0 を入れておきます。これを忘れると正しく数えられません。
📱 b16「forで出せる枚数を数える」 に挑戦しましょう！
◆
コラム
ループは「くり返しの自動化」―― コンピュータが得意なこと

第4章で学んだループ（くり返し）は、コンピュータが人間より圧倒的に得意なことです。何千件、何万件でも、文句を言わず同じ処理をくり返せます。

📧 例1：メールの一斉送信

i を 1 から 会員数 まで 1 ずつ増やしながら， ｜　会員[i] にメールを送る を繰り返す

「全員に1通ずつ送る」を自動化しています。大富豪で「手札を1枚ずつ全部調べた」のと同じ構造です。

🛒 例2：買い物カゴの合計金額

goukei ← 0 i を 1 から 品数 まで 1 ずつ増やしながら， ｜　goukei ← goukei ＋ 値段[i] を繰り返す

goukei ← 0 で初期化してから足していくのは、b16 で cnt ← 0 から数えたのと全く同じパターンです。

📊 まとめ：ループの2大パターン

パターン使う変数大富豪での例
数える（集計）カウンタ（0から＋1）b16 出せる枚数
探す（検索）フラグ（0→値）b17 出せる最弱札

「数える」と「探す」――この2つを押さえれば、ループの応用範囲が一気に広がります。

CHAPTER 5
前判定ループ
〜の間，くり返す ・ 条件で回数が変わるループ

📖 5-1　前判定ループ ―― 条件が真の間くり返す

forループは「1から要素数まで」と回数が決まったくり返しでした。一方、前判定ループは「ある条件が成り立っている間ずっとくり返す」という、回数が状況で変わるループです。

前判定ループの形：
条件 の間，
｜　処理
を繰り返す

「前判定」とは、くり返しの前に条件をチェックするという意味です。条件が最初から偽なら、処理は1回も実行されません。

📖 5-2　カウンタを自分で動かす

forループは i を自動で増やしてくれましたが、前判定ループでは自分でカウンタを増やす必要があります。

# ① ループの前でカウンタを初期化 i ← 1 # ② 条件が真の間くり返す i ≦ 要素数[手札] の間， ｜　# ③ ループの中で必ず i を進める（無限ループ防止） ｜　i ← i ＋ 1 を繰り返す
⚠️ 無限ループに注意：前判定ループでは、ループの中で条件がいつか偽になるようにしないと、永遠に止まりません。カウンタ i を必ず進めましょう。

📖 5-3　「探す」ループに向く

前判定ループは「条件を満たすものが見つかるまで進める」という探索にぴったりです。見つかったらループを抜け、抜けた後の i の値で結果を判断します。

📱 b19「前判定ループで出せる札を探す」 に挑戦しましょう！
◆
コラム
「回数が決まらないくり返し」―― forとwhileの使い分け

くり返しには2種類あります。回数が分かっているときはforループ、「○○になるまで」のように回数が読めないときは前判定ループ（while）が向いています。

🎮 例1：ゲームの「ライフがある間」

life ≧ 1 の間， ｜　ゲームを続ける を繰り返す # ライフが0になったら自動で終了

何ターン続くかは事前に分かりません。「条件が成り立つ間」だけ回す前判定ループの出番です。

🔍 例2：データを「見つかるまで」探す

i ← 1 データ[i] ≠ 探す値 の間， ｜　i ← i ＋ 1 を繰り返す

大富豪で「場より強い札を見つけるまで」進めた b19 と同じ、探索（リニアサーチ）です。

📊 まとめ：使い分けの目安

状況向いているループ
「全部の手札を調べる」forループ（回数＝枚数）
「見つかるまで探す」前判定ループ（while）
「条件が成り立つ間ずっと」前判定ループ（while）

どちらも「くり返し」ですが、回数の決まり方がちがいます。場面に合わせて選びましょう。

CHAPTER 6
後判定ループ・中断する
中断する ・ 繰り返し〜になるまで実行する

📖 6-1　中断する ―― ループを途中で抜ける

第4章のforループでは、目的のカードが見つかっても最後まで回り続けていました。中断する を使うと、その場でループをすぐ抜けられます。

中断するの使い方：
i を 1 から 要素数[手札] まで 1 ずつ増やしながら，
｜　もし 見つかった ならば
｜　｜　found ← 手札[i]
｜　｜　中断する　# ここでループを抜ける
｜　を実行する
を繰り返す

b17 では found ＝ 0 という条件で「最初の1枚だけ記録」しましたが、中断する なら見つけた瞬間に止まるので、その条件すらいりません。よりシンプルになります。

📖 6-2　後判定ループ ―― 必ず1回は実行する

前判定ループ（第5章）は「先に条件をチェック」しました。後判定ループはその逆で、処理を1回実行してから条件をチェックします。だから条件に関係なく最低1回は必ず実行されます。

後判定ループの形：
繰り返し，
｜　処理
を，条件 になるまで実行する

「○○になるまでくり返す」という書き方です。条件が真になったらループを抜けます（前判定とは真偽が逆なので注意）。

⚠️ 前判定との違い：前判定は「条件が真の間続ける」、後判定は「条件が真になるまで続ける（＝偽の間続ける）」。判定のタイミングと真偽が逆です。
📱 b18「見つけたらループを中断する」 に挑戦しましょう！
◆
コラム
3つのループを整理する ―― for・while・do-while

ここまでで3種類のくり返しを学びました。プログラミング言語では順に for / while / do-while と呼ばれます。それぞれの「止まり方」を整理しましょう。

🔁 3つのループの比較

種類本書での呼び方判定のタイミング最低実行回数
forforループ（4章）回数で決まる0回
while前判定ループ（5章）実行する前0回
do-while後判定ループ（6章）実行した後1回

🏧 例：ATMの暗証番号入力

繰り返し， ｜　暗証番号を入力させる を，番号 ＝ 正解 になるまで実行する

「まず1回は必ず入力させたい」ので後判定（do-while）が自然です。b20 で最後まで走らせたのと同じ構造です。

⚡ 中断する（break）はどのループでも使える

b18 で使った 中断する は、ループを途中で抜ける万能の道具です。「見つかったらもう用はない」探索では、最後まで回さず途中で抜けることで処理が速くなります。多くの言語で break と呼ばれます。

📖 7-1　配列とは ―― 番号つきの箱の列

要素数[配列名] で、その配列に入っている個数が分かります。手札 以外にも使えます。

# 配列Tの1番目に手札[1]をコピー T[1] ← 手札[1] T[2] ← 手札[2]

配列は「ランクを添字にして数を数える」のにも使えます。たとえば C[5] に「ランク5が何枚出たか」を入れる、という使い方です。

# ランク r のカウントを1増やす C[r] ← C[r] ＋ 1

これは「番号で仕分けして数える」強力なテクニックで、集計（ヒストグラム）と呼ばれます。

⚠️ 使う前に初期化：集計用の配列は、数え始める前にすべての要素を 0 にしておきます（例: j を 3 から 15 まで 1 ずつ増やしながら，｜　C[j] ← 0 を繰り返す）。
※このアプリでは、まだ値を入れていない変数・配列要素は 0 として読めますが、共通テストの DNCL では初期化を書くのが正式です。b16 で「数える変数は最初に 0」と学んだのと同じルールです。
