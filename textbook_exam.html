■ 表記が3種類ある理由
DNCL には「公式仕様（DNCL.pdf）」「アプリの表記」「試験での実際の表記」の3種類があります。
公式仕様書自体に「記述内容に従わない形式で出題することもあります」と明記されており、試験ごとに注記で表記が説明されます。
大切なのは表記を丸暗記することではなく、アルゴリズムの意味を読み取る力です。

0-2　DNCL とは

大学入試センターが定めた「共通テスト用プログラム表記（DNCL）」です。実際のプログラミング言語ではなく、日本語ベースの疑似コードです。

公式仕様（DNCL.pdf 2022）の特徴
・代入：←
・演算子：+-><% は半角、×÷ は全角
・if 文は を実行する で閉じる
・配列添字は 1 始まり
試験での表記例（令和3・6年度）
・代入：=（半角等号）
・if 文末：コロン :
・表示：表示する(値)
・演算子は半角が多い
試験に出やすいポイント
試験の第1ページまたは最初の問題の直前に「この問題で使用するプログラムの表記」が載っている。最初の2分でここを読み、その試験固有の表記を把握すること。表記の違いで内容は変わらない。
✅ 自己チェック（第 0 章）
順次・分岐・反復の3構造をそれぞれ1文で説明できる
DNCL は公式仕様と試験で表記が異なる場合があることを説明できる
試験開始時に「表記の注記」を読む習慣をつけた

1-3　インクリメント・デクリメント構文（仕様特有）

DNCL 仕様特有：インクリメント構文

DNCL 公式仕様では「変数を1増やす」を短縮形で書けます。

短縮形（仕様書記法）
kaunto を 1 増やす
gokei を ten 増やす
通常の代入（同じ意味）
kaunto ← kaunto + 1
gokei ← gokei + ten
試験では「kosu を 1 増やす」のような形で出題される場合があります。
試験に出やすいポイント
代入は「右辺 → 左辺」の方向。x = x + 1 は「x の現在値に 1 を加えた結果を x に入れる」。数学の等号とは意味が違う。試験では「このプログラムを実行後の x の値は？」という問いが多い。トレースが必須。
確認問題 ① 変数と代入
  1. 次のプログラムを実行後、x の値はいくつか。
    x = 3 / x = x * 4 / x = x - 2　→　
  2. gokei を ten 増やす を通常の代入で書き直せ。
  3. 「外部から ninzu を入力し、ninzu が 0 より大きければ表示する」プログラムを DNCL で書け。
✅ 自己チェック（第 1 章）
代入の3表記（← / ← / =）すべてで同じ意味だと確認できた

7-3　複数の代入を1行に並べる（仕様特有）

「，」で区切ると、左から順に実行される（同時ではない）

DNCL では、複数の代入文を「，」で区切って 1 行に並べることができます。公式仕様では 代入文は左から順に実行されます。「右辺を先にまとめて評価する」わけではない点に注意してください。

例 ― 合計に加えてからカウンタを進める
goukei ← goukei ＋ x, x ← x ＋ 1
　左の goukei ← goukei ＋ x を実行してから、
　右の x ← x ＋ 1 を実行する（左 → 右の順）
よくある誤解（値の交換）：a ← b, b ← a は値の交換にはなりません。左から順に実行されるため、まず a ← b で a が b の値になり、続く b ← a では a がすでに更新済みなので、a も b も同じ値（元の b）になってしまいます。
値を入れ替えるには一時変数を使います：w ← a, a ← b, b ← w（a=10, b=20 なら実行後は a=20, b=10）。
