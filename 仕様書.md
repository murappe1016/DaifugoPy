## 1. プロジェクト概要

**大富豪プログラミング (DaifugoPy)** は、トランプゲーム「大富豪」の戦略を DNCL（擬似言語）で記述して学ぶ、純粋なブラウザアプリ（HTML/CSS/JS）です。

- サーバーサイド不要。`index.html` をブラウザで開くだけで動作。
- 3つのモード: **基礎問題** / **戦略問題** / **フリーモード**

## 3. ゲームルール（実装上の定義）

| カード値 | 数字 | 備考 |
|---------|------|------|
| 3〜10   | そのまま | 最弱=3, 最強=15($) |
| J       | 11 | |
| Q       | 12 | |
| K       | 13 | |
| A       | 14 | |
| $（ジョーカー）| 15 | 最強 |

- `fieldStrength`: 場のカードの強さ（空き場=0）
- `fieldCount`: 場に出ているカード枚数（0/1/2）
- 手札は弱い順にソート済み（`playerHand` 配列）
- 8が出ると「8切り」（場がリセット）

## 4. DNCL インタープリター（dncl.js）

### 対応構文

```
変数 ← 式                          # 代入
「文字列」 と 式 を表示する          # 表示
もし 条件 ならば | 処理 | を実行する
もし ... ならば | ... | を実行し，そうでなければ | ... | を実行する
変数 を 初期値 から 終了値 まで 差分 ずつ増やしながら，| 処理 | を繰り返す
条件 の間，| 処理 | を繰り返す
繰り返し，| 処理 | を，条件 になるまで実行する
関数 名前(引数) を | 処理 | と定義する
返す(式)
出す(式, 式)   # カードを出す
パスする
中断する       # break
```

### ゲーム変数（読み取り専用）

| 変数名 | 内容 |
|-------|------|
| `場の強さ` | fieldStrength |
| `場の枚数` | fieldCount |
| `手札[i]` | playerHand[i-1]（1始まり） |
| `要素数[手札]` | playerHand.length |
| `捨札[i]` | allDiscard[i-1]（1始まり） |
| `要素数[捨札]` | allDiscard.length |

### インタープリターの呼び出し

```javascript
const interp = new DNCLInterpreter(stateObject);
const result = interp.run(codeString);
// result: { action: 'play'|'pass', rank, count, displayMessages: [] }
```

`stateObject` の形式:
```javascript
{
  fieldStrength: 0,
  fieldCount: 0,
  playerHand: [3, 7, 10],   // 弱い順にソート済み
  allDiscard: [],
  opponentHandSize: 3,
}
```

## 5. problems.json の構造

```json
{
  "problems": [
    {
      "id": "b01",
      "category": "basic",
      "title": "先頭のカードを出す",
      "subtitle": "つかうもの: 手札 / 出す",
      "daifugoTitle": "手札の最弱カードを出す",
      "difficulty": 1,
      "activeSection": 0,
      "activeRefs": ["h", "e", "d"],
      "description": "プレーンテキストの問題文",
      "descriptionHtml": "<HTML形式の問題文>",
      "hints": [
        "ヒント1",
        "ヒント2",
        "答え:\nコード"
      ],
      "testCases": [
        {
          "label": "手札 3,7,10 → 3 を出す",
          "state": {
            "fieldStrength": 0,
            "fieldCount": 0,
            "playerHand": [3, 7, 10],
            "allDiscard": [],
            "opponentHandSize": 3
          },
          "expected": {
            "action": "play",
            "rank": 3,
            "count": 1
          }
        }
      ]
    }
  ]
}
```

### expected の形式

| パターン | フィールド |
|---------|-----------|
| カードを出す | `{ "action": "play", "rank": N, "count": 1または2 }` |
| パスする | `{ "action": "pass" }` |
| 表示する（のみ） | `{ "action": "pass", "display": "表示すべき文字列" }` |

`display` が指定されている場合、`displayMessages` に指定文字列が含まれていれば正解。

### activeSection

コードエディターは3セクション（場の枚数=0/1/2）に分かれており、`activeSection` で編集可能なセクションを指定する。

| activeSection | 対応するコードブロック |
|--------------|----------------------|
| 0 | `もし 場の枚数 ＝ 0 ならば`（空き場） |
| 1 | `そうでなくもし 場の枚数 ＝ 1 ならば`（1枚出し） |
| 2 | `そうでなくもし 場の枚数 ＝ 2 ならば`（2枚出し） |

**重要**: `activeSection` が `null`/`undefined` だと全セクションがロックされる。必ず整数で設定すること。

### activeRefs（サイドバー表示キー）

| キー | 表示内容 |
|-----|---------|
| `f` | 場の強さ |
| `n` | 場の枚数 |
| `h` | 手札[＜式＞] |
| `e` | 要素数[＜変数＞] |
| `s` | 捨札[＜式＞] |
| `i` | もし〜ならば |
| `I` | もし〜ならば/そうでなければ |
| `w` | 前判定ループ |
| `r` | 後判定ループ |
| `l` | forループ |
| `F` | 関数定義 |
| `d` | 出す(式, 式) |
| `p` | パスする |
| `b` | 中断する |
| `R` | 返す(式) |
| `t` | ＜式＞ を表示する |

## 8. 採点フロー（テストケースUI）

採点は1ケースずつ手動で進める対話型フローです。

### フロー

```
「採点する」クリック
  → テストケース1の場・手札を表示（対戦スタート前の状態）
  → 「▶ 対戦スタート」クリック
      → コード実行
      → 表示するの出力あり → 中央モーダルにポップアップ（OK で閉じる）
      → 結果表示（✅/❌ + あなたの表示 vs 正解）
      ✅ 正解 → 「次のケースへ →」ボタン → 次ケースの状態表示へ
      ❌ 不正解 → 「修正して採点し直す」メッセージ → 止まる
  → 全ケース通過 → 🎉 クリア（問題一覧に ○ が付く）
```

### 関連関数（js/ui.js）

| 関数名 | 役割 |
|-------|------|
| `scoreCode()` | 採点ボタン押下時の処理（前提チェック→initScoringShell） |
| `initScoringShell(total)` | HTMLシェル構築、モーダルOKイベント登録 |
| `setupScoringCase(idx)` | ケースの場・手札表示、対戦スタートボタン設置 |
| `runScoringCase(idx)` | コード実行、displayMessages処理 |
| `showScoringResult(idx, r)` | 結果（pass/fail）の表示と次のボタン制御 |
| `showScoringFinal(total)` | 全ケース通過後のクリア表示 |

### 表示するモーダル（HTML）

```html
<div id="display-modal" class="dm-overlay hidden">
  <div class="dm-box">
    <div class="dm-title">📺 表示する の出力</div>
    <div id="display-modal-lines" class="dm-lines"></div>
    <button id="display-modal-ok" class="primary-btn dm-ok-btn">OK</button>
  </div>
</div>
```

## 10. 重要な実装上の注意点

### lockSections のフォールバック

`activeSection` が `null`/`undefined` だと全セクションがロックされる。対策として以下を追加済み：

```javascript
function lockSections(activeN) {
  if (activeN == null) activeN = 0;  // フォールバック
  ...
}
```

### buildFullCodeForScore の構造

採点時、ユーザーのコードは以下の形に展開される：

```
もし 場の枚数 ＝ 0 ならば
  [codes[0] または 空]
を実行し，そうでなくもし 場の枚数 ＝ 1 ならば
  [codes[1] または 空]
を実行し，そうでなくもし 場の枚数 ＝ 2 ならば
  [codes[2] または 空]
を実行し，そうでなければ
｜　パスする
を実行する
```

**テストケースの `fieldCount` と `activeSection` は必ず一致させること。**

例: `activeSection=1` の問題のテストケースは全て `fieldCount=1` にする。

ファイルを編集しても `/tmp/daifugo-preview/` に同期しないと反映されない（上記コマンド参照）。
