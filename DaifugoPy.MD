大富豪戦略デバッグ・対戦アプリ 開発仕様書

1. プロジェクト概要

ユーザーが共通テスト手順記述標準言語（DNCL）風の記法で「大富豪の戦略」を記述し、特定のカード状況を指定して1手ずつ動作を検証（デバッグ）できる学習・検証用Webアプリ。

2. アプリのゴール

「自分が考えた戦略（ロジック）が、想定した通りのカードを出しているか」を、ランダム性に頼らず確実に100%の再現性をもって確認すること。

3. メインフロー

カード設定: 3〜2（13種）から、自分と相手の手札を自由に選んで固定する。

戦略コーディング: ショートカット機能（@22等）を活用し、DNCL記法で3つの状況別の戦略を書く。

対戦デバッグ: 「次へ」ボタンを押し、1手ごとにAIの判断（出すカードの予報）と実行結果を確認する。

再戦・修正: 勝敗が決した後は同じ手札で即座に再開可能。「変更」でカードを選び直す。

4. 機能要件

4.1. カード指定機能（手札の固定）

選択ルール: 各数字ランクにつき、自分と相手を合わせて最大2枚まで使用可能。

UI: 12（13）枚のカード一覧からクリックで自分・相手に振り分ける。

リセット: 試合終了後、自動的に「最初と同じ手札」が配られた状態に戻る。

4.2. 戦略記述エディタ

編集対象: 場の枚数（0, 1, 2）に応じた3つの関数ブロック。

入力補助（ショートカット）:

@番号 を入力すると、対応する変数名や制御構文が展開される。

例：@22 → 強さ を 場の強さ ＋ 1 から 13 まで 1 ずつ増やしながら繰り返す：

4.3. デバッグ実行エンジン

ユーザー先行: 基本的にユーザーが親からスタート。

相手AI: 常に「場に出せる中で最も弱いカードを出す」固定ロジック。

ステップ実行: 「次へ」ボタンによる手動進行。

動作予報: 実行前に「次に出す予定のカード」をハイライト表示。

6. 戦略記述の例（DNCL形式）

ユーザーが記述する「出せる最弱を出す」ロジックの記述例です。

出したフラグ ＝ 0

もし 場の枚数 ＝＝ 0 ならば：
|   # 親：手札で一番弱いカードを1枚出す
|   出したランク ＝ 手札[0]
|   出す(出したランク, 1)
|   出したフラグ ＝ 1

そうでなければ もし 場の枚数 ＝＝ 1 ならば：
|   # シングル：場の強さより1つ強いランクから順に探す
|   強さ を 場の強さ ＋ 1 から 13 まで 1 ずつ増やしながら繰り返す：
|   |   枚数 ＝ 0
|   |   i を 0 から (手札の要素数 － 1) まで 1 ずつ増やしながら繰り返す：
|   |   |   もし 手札[i] ＝＝ 強さ ならば：
|   |   |   |   枚数 ＝ 枚数 ＋ 1
|   |   |   もし 終わり
|   |   繰り返し 終わり
|   |
|   |   もし 枚数 ≧ 1 ならば：
|   |   |   出す(強さ, 1)
|   |   |   出したフラグ ＝ 1
|   |   |   中断する
|   |   もし 終わり
|   繰り返し 終わり

もし 出したフラグ ＝＝ 0 ならば：
|   パスする
もし 終わり
