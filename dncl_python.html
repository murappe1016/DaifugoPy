DNCL ↔ Python 対応表

大富豪プログラミングの表記（共通テスト用プログラム表記に準拠）を、Python の書き方と並べて理解する。
いちばん大事な違い：添字（番号）の起点。 DNCL の 手札[1] は1番目＝Python では hand[0]。 DNCL は1始まり、Python は0始まり。手札[要素数[手札]]（最後）は Python で hand[len(hand)-1] または hand[-1]。

意味DNCL（このアプリ）Python
変数に代入n ← 5n = 5
値を表示n を表示するprint(n)
文字列と値をつなぐ「合計は、」 と n を表示するprint("合計は、", n)
1ずつ増やすi ← i ＋ 1i = i + 1 （i += 1）

注意：DNCL の ÷ は答えの小数点以下を切り捨てる整数の割り算。Python では // が対応（7 // 2 == 3）。

⑤ 繰り返し

DNCL（for）
i を 1 から 要素数[手札] まで
　　　1 ずつ増やしながら，
｜　合計 ← 合計 ＋ 手札[i]
を繰り返す
Python（for）
for i in range(1, len(hand)+1):
    total = total + hand[i-1]
# よりPythonらしく:
# for c in hand: total += c
DNCL（前判定 while）
i ≦ 要素数[手札] の間，
｜　i ← i ＋ 1
を繰り返す
Python（while）
while i <= len(hand):
    i = i + 1
後判定ループ（繰り返し，… を，条件 になるまで実行する）は「必ず1回実行」する形。Python に専用構文はなく while True: … if 条件: break で書く。中断する ＝ break。

⑥ 関数

DNCL
関数 出せるか(c) を
｜　もし c ＞ 場の強さ ならば
｜　｜　返す(1)
｜　を実行する
｜　返す(0)
と定義する
Python
def can_play(c):
    if c > field:
        return 1
    return 0

⑦ まるごと翻訳例（b16：出せる枚数を数える）

DNCL
cnt ← 0
i を 1 から 要素数[手札] まで
　　　1 ずつ増やしながら，
｜　もし 手札[i] ＞ 場の強さ ならば
｜　｜　cnt ← cnt ＋ 1
｜　を実行する
を繰り返す
「出せるカードは、」 と cnt を表示する
Python
cnt = 0
for i in range(1, len(hand)+1):
    if hand[i-1] > field:
        cnt = cnt + 1
print("出せるカードは、", cnt)
